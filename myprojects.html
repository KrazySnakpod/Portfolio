My Projects

A list of all the projects I've worked on, from Game Jam submissions to University projects, as well as personal projects that I'm still working on.

Game Jam submissions

Clownery

Created during the Global Game Jam 2024 with the theme of "Make Me Laugh", Clownery is a multiplayer game with two main phases - a daytime phase, where the players perform on stage and use skills to gather money, and a nighttime phase, where players must stealthily break into a series of secured areas and steal as much as possible, using their money to purchase useful tools to simplify the process.

  • Created in Unity
  • Written in C#

The Traveller

Created during the Ryan Laley Games Summer Jam 2023 with the theme of "Contrast", The Traveller is a short game in which you play as a time-hopping traveller attempting to breach the castle. The traveller cannot attack, and as such must utilize their time-hopping abilities to manoeuvre past any enemies they encounter.

  • Created in UE5

Unfinished Turn-Based RPG

Inspired by numerous turn-based RPG games, I wanted to find out how the combat system in these games could be implemented. The player is able to use a basic attack, use skills costing HP or MP, and inflict status ailments that deal damage over time. Each entity has their own stat table, determining stats such as health, physical attack and speed dependent on their current level. This project started in Summer 2023 as my first Unreal Engine project.

  • Created in UE5
  • Written in Blueprint

University projects

Survival Horror AI

A short demo created as part of my dissertation project, utilizing a GOAP system and the A* pathfinding algorithm to determine a course of events for each enemy. The player navigates a dark area while the enemies navigate the area, responding to sight and audio cues. If the player is spotted, the enemy will begin to chase the player until either the player is defeated or the enemy loses a player, in which it will decide the next best course of action.

  • Created in UE5
  • Written in Blueprint & C++
>

Rob O'Mann

Rob O'Mann is a mobile game where the player takes on the role a robot who wants to learn more about humanity and learn how to become human, named Rob O'Mann. A stealth game, Rob must navigate past enemies stealthily and interact with the environment to gather intel. With enough intel, Rob can complete the level, where his grade is determined by how long the level took and whether he used any items or a disguise. This game was created as part of a team, and I was primarily responsible for the movement system and adapting it to work on mobile devices, among other things such as points and the camera.

  • Created in Unity
  • Written in C#
  • Available on mobile devices
>

Visual Simulation

In the Visual Simulation module, our task was to create an interactive and engaging visualization of a set of data in virtual reality, with the ability for the data to be changed and reflected in the game. As a group, we decided that visualizing the data set using a series of roads in the style of the game Crossy Road would be effective for this task, with the number of cars on the roads representing the data. In this project, I was responsible for the items system, the VR setup and much of the main backend systems, such as the ability to dynamically change the world by changing just the data set.

  • Created in Unity
  • Written in C#
  • Runs on virtual reality headsets
>

Multiplayer Asteroids clone

For this project, we were tasked with making a clone of the Atari game Asteroids which allowed for online multiplayer connectivity using a Server and multiple clients. The codebase originally provided to us contained libraries which enabled a UDP socket connection, however it was up to us to establish network connectivity for the objects. The final game is a near identical copy of Asteroids, with network functionality added enabling multiplayer, however the score is not currently tracked.

  • Written in C++
  • Utilizes SDL2 & SDL2_image
  • Requires a running Server instance
>

Game Engine Development

The task for this module was to identify the components of a game engine, and to use a combination to create a simplistic clone of the game Forsaken. The project created for this module is not a complete game, however it does demonstrate some of the features of Ogre3D, featuring lighting, multiple cameras, and fog, and also demonstrates some functionality of Bullet, featuring collisions, both using simple objects and meshes to define a collision zone. Finally, there is a small amount of AI functionality.

  • Written in C++
  • Utilizes Ogre3D for rendering & Bullet for physics
>

Level Design & Game Assets
